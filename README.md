## 🚀 快速开始

### 在线体验（无需安装）
👉 [https://mrx666-hub.github.io/MRXTaskMaster/](https://mrx666-hub.github.io/MRXTaskMaster/)

### 方式一：本地直接使用
```bash
# 1. 克隆仓库
git clone https://github.com/mrx666-hub/MRXTaskMaster.git

# 2. 进入项目目录
cd MRXTaskMaster

# 3. 用浏览器打开 index.html 即可使用
open index.html # macOS
start index.html # Windows
xdg-open index.html # Linux
```

### 方式二：部署到自己的GitHub Pages
```bash
# 1. Fork 本仓库到你的 GitHub 账号
# 2. 进入仓库 Settings → Pages
# 3. Source 选择 main 分支，点击 Save
# 4. 访问 https://<your-username>.github.io/MRXTaskMaster 即可使用
```

### 方式三：部署到自己的服务器
只需将所有文件上传到任意静态文件服务器即可使用，无需后端支持。

## 🛠 技术实现

### 架构设计
本项目采用经典 MVC 架构模式，代码结构清晰，易维护：
- **Model**: `tasks` 数组 + localStorage 存储逻辑，负责数据管理
- **View**: `renderTable()` + 终端输出渲染函数，负责界面展示
- **Controller**: 命令解析器 + 事件处理逻辑，负责业务逻辑处理

## 📱 PWA 部署指南

本项目支持直接部署为 PWA（渐进式 Web 应用），可安装到桌面，离线使用：

### 步骤 1：创建 manifest.json 文件
在项目根目录创建 `manifest.json` 文件：
```json
{
  "name": "MRXTaskMaster",
  "short_name": "TaskMaster",
  "description": "命令行任务管家，高效管理你的待办事项",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#0f172a",
  "theme_color": "#3b82f6",
  "icons": [
    {
      "src": "icon.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "icon-512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ]
}
```

### 步骤 2：添加 PWA 支持代码到 index.html
在 `index.html` 的 `<head>` 标签中添加：
```html
<link rel="manifest" href="manifest.json">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="default">
<meta name="apple-mobile-web-app-title" content="TaskMaster">
<link rel="apple-touch-icon" href="icon.png">
<script>
  if ('serviceWorker' in navigator) {
    window.addEventListener('load', () => {
      navigator.serviceWorker.register('/sw.js');
    });
  }
</script>
```

### 步骤 3：创建 Service Worker 文件
在项目根目录创建 `sw.js` 文件，实现离线缓存功能：
```javascript
const CACHE_NAME = 'taskmaster-cache-v1';
const ASSETS = [
  '/',
  '/index.html',
  '/style.css',
  '/script.js'
];

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then((cache) => cache.addAll(ASSETS))
      .then(() => self.skipWaiting())
  );
});

self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then((cacheNames) => {
      return Promise.all(
        cacheNames.filter((name) => name !== CACHE_NAME)
          .map((name) => caches.delete(name))
      );
    })
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request)
      .then((response) => response || fetch(event.request))
  );
});
```

### 步骤 4：添加应用图标
准备 192x192 和 512x512 像素的 PNG 图标，命名为 `icon.png` 和 `icon-512.png` 放在项目根目录。

完成以上步骤后，重新部署项目，用户访问时就可以选择"安装应用"到桌面了。

## 🤝 贡献指南

欢迎提交 Issue 和 Pull Request 来改进这个项目！

### 开发环境要求
- 无需任何构建工具，直接用浏览器打开 `index.html` 即可开发
- 建议使用 VS Code + Live Server 插件进行开发，获得更好的开发体验

- `feat: 新功能` - 添加新的功能特性
- `fix: 修复 bug` - 修复已知的问题和错误
- `docs: 文档更新` - 更新文档、README 等
- `style: 样式调整` - 调整 UI 样式、代码格式等（不影响功能）
- `refactor: 代码重构` - 代码重构，不改变功能逻辑
- `perf: 性能优化` - 提升代码性能、运行效率
- `chore: 构建/工具` - 构建流程、工具配置等修改

### 开发建议
1. Fork 本仓库到自己的账号下

3. 提交你的修改：`git commit -am 'feat: 添加新功能'`
4. 推送到分支：`git push origin feature/your-feature-name`
5. 提交 Pull Request，等待审核
